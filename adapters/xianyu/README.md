# 闲鱼成交侧 adapter（B 档：human-in-the-loop, read-only）

给 cheat-on-money skill 提供**真实成交侧数据**(在卖什么、什么价、多少人想要),补上通用网页检索拿不到一手平台数据的短板。

## 它是什么 / 不是什么
- ✅ 是:读你**已登录、已搜索好**的闲鱼当前页面,把商品卡片提成结构化数据。
- ❌ 不是:它**不自动登录、不自动搜索、不自动翻页、不模拟点击**。导航、搜索、过滑块全是你(真人)在做。
- 为什么这么设计:真人操作不触发风控、不像批量爬虫,在平台 ToS / 法律上最站得住(详见根 README 的风险说明)。

## 用法(3 步)
```bash
cd adapters/xianyu
npm install                      # 只装 playwright-core,不下载浏览器(已装过可跳过)

./launch-chrome.sh               # 1) 起一个【有界面】Chrome(无头会被反爬挡!)
                                 # 2) 在窗口里【扫码登录闲鱼】(不登录看不到结果),窗口留着
node read-xianyu.mjs "AI头像" 9222   # 3) 在你登录态下搜该词、读结果 → JSON(标题/价格/想要数/链接)
```
输出的 JSON 直接喂给 `money-find` 当成交侧信号。

## 实测验证过的事实
- ✅ **有界面 Chrome 能过闲鱼反爬**(已实测进到真实搜索页)。
- ❌ **无头 Chrome 被挡**("非法访问,请使用正常浏览器") → 脚本不用无头。
- ⚠️ **匿名看不到结果**(页面停在"加载中"+提示登录) → **必须先扫码登录**,这步只能你做。
- ✅ 提取启发式已用 mock 页验证 **3/3** 正确。

## 已知限制(诚实标注)
- **真实结果页的选择器还没在登录态下校准过**(作者无法替你扫码登录)。**首次跑若条数为 0 或不对,把输出的 `diagnostics` 发回来**(尤其 `blocked`/`maybeNeedLogin`/`sampleRawText`),几分钟可调准。
- **read-only、低频**:不自动翻页/下单/私聊。要更多就换个词再跑。
- **闲鱼改版会让选择器腐烂**,需偶尔微调。
- **BOSS 不走这个**:招聘 JD 含个人信息、风控更凶 → 手动复制 JD 贴给 skill,不自动化。

## 风险红线(和 cheat-on-money skill 自己的反诈一致)
- 低频、read-only、真人操作 → 风险可控;**别改成自动批量爬**(触发风控/封号,且违反 ToS,碰个人信息保护法,与反诈 rubric A6 冲突)。
