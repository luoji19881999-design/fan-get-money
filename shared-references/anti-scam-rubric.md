# 反诈 Rubric —— "这机会是真能用的吗"判定标准

> 这是整个 cheat-on-money skill 的核心。任何机会在被推荐给用户前，必须过一遍本文档。
> 原则：**可信度靠机制，不靠内容质量。** 一个说得天花乱坠的机会，命中红线照样淘汰。

---

## 第一性原理（先记住这个，比任何清单都重要）

真实的赚钱机会，钱的流向永远说得清：

> **某个真实客户，因为某个真实需求，付钱买你用 AI 做出来的产出/服务。**

凡是符合下面任一描述的，本质上都不是"赚钱"，是"被赚钱"：

- 钱来自"系统/平台/项目本身"，而不是某个有需求的客户 → **假**
- 你要先付钱才能开始赚钱 → **假**
- 你的收入主要来自"拉来更多和你一样的人" → **传销**
- 这个机会卖的东西就是"教你怎么搞钱" → **套娃割韭菜**

判定时，先问一句话：**"谁付钱？为什么付？付多少？"** 答不上来或答得含糊 → 直接降级到"存疑"以下。

---

## A. 硬红线（命中任意一条 = 高危，直接淘汰，不进入推荐）

| # | 红线 | 典型话术/形态 |
|---|---|---|
| A1 | **入场付费** | 交押金/培训费/会员费/买课/买设备/买"激活码"才能开工 |
| A2 | **收益承诺** | "日入过千""月入过万""轻松躺赚""稳赚不赔""被动收入" |
| A3 | **拉人头返佣** | 收入主要来自发展下线、邀请好友、层级分销（传销结构） |
| A4 | **资金/账户经手** | 刷单、垫付、过账、跑分、出租/出借身份证·银行卡·收款码（**涉嫌违法犯罪**） |
| A5 | **卖"搞钱课"套娃** | 核心产品就是"我教你怎么用 AI 赚钱"的课程/社群 |
| A6 | **要求做违法/灰产** | 代写论文造假、虚假宣传、伪造内容、规避平台审核、爬取倒卖个人信息 |

> 命中 A4/A6 的，除了淘汰，还要**明确提醒用户这可能违法**，不要含糊。

## B. 存疑信号（命中需额外验证，不直接淘汰，但必须如实告知用户）

| # | 信号 | 为什么可疑 |
|---|---|---|
| B1 | 找不到任何第三方真实收款证据 | 网上只有招募贴、没有"真收到钱"的独立反馈 |
| B2 | 报酬远高于市场行情 | 同类活儿别人报价的数倍 → 通常是钓饵 |
| B3 | 只用微信/QQ 私聊，绕开平台担保 | 没有平台兜底，纠纷无解，跑路成本为零 |
| B4 | 催促"名额有限""今天不交明天没了" | 制造稀缺逼你冲动决策 |
| B5 | 要求先加群、先做"试岗任务" | 试岗常是免费白嫖或刷单前奏 |
| B6 | 金主/雇主信息查无此人 | 公司搜不到、营业执照对不上、客服话术含糊 |

## C. 验证动作（对"存疑"或用户想确认的机会，实际去查）

执行时**用 WebSearch / WebFetch 实时查**，不要凭记忆下结论：

1. **搜负面**：`<平台名/项目名> 骗局` / `<平台名> 靠谱吗` / `<平台名> 被骗 投诉`
2. **查实体**：平台是否有正规备案、是否走平台担保交易、公司是否能在企查类信息中找到
3. **找真实反馈**：在小红书/知乎/贴吧/黑猫投诉搜真实用户经历，区分"软文"和"真人吐槽"
4. **验证收款路径**：是否存在"小成本试水"——能先接 1 单最小的活，确认**钱真能到账**，再投入时间

> 验证结论要落到一句话：**"建议先用 ___ 这个最小动作验证能不能收到第一笔钱，再决定要不要投入。"**

---

## C′. 时效性核查（强制，和反诈同等重要）

平台规则、佣金、接单渠道、价格变化极快。**一条过时的信息，哪怕本身不是骗局，也会让人按已经失效的玩法去做，等于白费时间——这同样是"不能用"。**

执行时**必须做到**：

1. **查每条来源的发布日期。** 引用任何渠道/价格/平台玩法前，先确认它的发布时间。
2. **时效降权规则**（以当前年份为准）：
   - 12 个月内：可用，正常引用。
   - 12–24 个月：**标注"可能过时"**，且必须再找一条近 12 个月的来源交叉验证，否则降级到 ⚠️。
   - 超过 24 个月：**默认失效**，不得作为"现在怎么做"的依据，除非有近期来源印证它仍成立。
3. **检索时主动加年份**（如 `2026`、`最新`、`今年`），并优先点开近期结果；老结果只用来理解"这个方向大概是什么"，不用来定"现在具体怎么做"。
4. 呈现给用户时，**每条关键信息标来源年份**，并整体注明"以下基于 <今天日期> 的检索"。

> 一句话：**反诈管"是不是真的"，时效管"现在还成不成立"。两关都过，才叫真能用。**

## D. 打分输出格式（money-verify 用）

对每个机会输出：

```
机会：<名称>
钱从哪来：<谁付钱/为什么/大概多少>
硬红线：<命中的 A 编号，或"无">
存疑信号：<命中的 B 编号，或"无">
信息时效：<关键来源的发布年份；是否做了近 12 个月交叉验证；有无过时风险>
判定：高危 ❌ / 存疑 ⚠️ / 可行 ✅
验证第一步：<一个今天就能做、低成本、能确认钱能到账的动作>
诚实收入预期：<前期大概多久、多少；明确说明这是"时间换小钱+攒作品"还是有快钱>
```

判定规则：
- 命中任一 A → **高危 ❌**
- 命中任一 B 且未通过 C 验证 → **存疑 ⚠️**
- 无 A、B 已验证清楚、钱的流向说得清 → **可行 ✅**

---

## E. 期望管理（每次推荐都要带的"清醒剂"）

- 绝大多数 AI 兼职前期是 **"用时间换小钱 + 攒作品集 + 攒客户口碑"**，不是躺赚。
- AI 不会替你赚钱，AI 只是**让你某个技能的产出效率变高**，能不能变现仍取决于有没有真实客户。
- "门槛越低、越不需要技能、越说能快速来钱"的机会，竞争越惨烈、越可能是坑。**护城河来自你的技能或垂类积累，不来自项目本身。**
- 真能用的判断标准不是"听起来能赚多少"，而是"我能不能用最小成本验证它真的有人付钱"。
