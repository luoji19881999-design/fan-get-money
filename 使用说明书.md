# fan-get-money 使用说明书

> 适用于 OpenAI Codex、Claude Code、Cursor、Gemini、Copilot、Windsurf、Hermes、WordBuddy 等 Vibe Coding 平台

---

## 六大命令

| 命令 | 做什么 | 什么时候用 |
|------|--------|------------|
| `搞钱初始化` | 5 个问题建立画像，判定 T0-T3 | 第一次用 |
| `帮我找机会` | 搜真实数据，输出机会列表 | 想找方向 |
| `验证 XX` | 过五层反诈，判高危/存疑/可行 | 不确定靠不靠谱 |
| `给我行动方案` | 生成两周具体计划+止损线 | 选定方向后 |
| `复盘` | 对比预测 vs 实际 | 做了一段时间 |
| `搞钱状态` | 查看画像、进度、下一步 | 随时 |

### 各平台触发

| 平台 | 怎么触发 |
|------|----------|
| Codex | `/fan-money-init` 等斜杠命令 |
| Claude Code / Gemini / Hermes / WordBuddy | 直接说中文，如"搞钱初始化" |
| Cursor / Copilot / Windsurf | 安装后自动生效，说中文即可 |

## 完整流程

```
搞钱初始化 → 帮我找机会 → 验证 XX → 给我行动方案 → 执行 → 复盘 → 搞钱状态
```

```
第 0 天

第 1 天
```

## 六命令详解

### 初始化（/fan-money-init）

问 5 个必要问题：技能 / 时间 / 资金 / 地区 / 能否露脸。不问你"兴趣爱好"。10 维度评估后判定 T0-T3 层级，生成 `.money-state.json`。

### 找机会（/fan-money-find）

根据 tier 选搜索策略。用适配器读闲鱼成交数据 + BOSS 直聘招聘数据 + WebSearch 交叉验证。每个机会标注来源、时效、收入预期。

### 反诈（/fan-money-verify）

五层递进：A 发现信号 → B 验证信号 → C 推理分析 → C' 24 分量化 → C" 关键词黑名单（命中 2 条直接高危）。

高危关键词：稳赚、躺赚、日入、包赚、月入过万、日结、宝妈、小白、保证收益、零风险

碰到以下情况直接跑：先交钱、加 QQ/微信私聊、试稿不给钱

### 行动方案（/fan-money-plan）

最低成本第一步 → 前两周天级动作 → 诚实收入预期 → 止损线。写入 `.money-state.json` 备案，供复盘对比。

### 复盘（/fan-money-retro）

对比预测 vs 实际。自动判断 tier 升级：T0→T1 需 1 条 AI 经验+收入转化；T1→T2 需熟练掌握 Cursor/Claude Code/Codex。沉淀到 `lessons.md` 和 `archive/`。

### 状态（/fan-money-status）

只读看板。画像、候选机会、当前进度、复盘历史、下一步建议。

## T0-T3 分层

| 层级 | 你是什么状态 | 适合方向 |
|------|-------------|----------|
| T0 | 零技能/零资金 | 平台任务、数据标注 |
| T1 | 会用 AI 工具 | 接单类技能服务 |
| T2 | 精通 AI + 有行业经验 | B端交付、出海、SaaS |
| T3 | 有团队/资本 | 产品化、规模化 |

## 常见问题

**Q: 零基础（T0）有用吗？**
A: 有用。T0 阶段最重要的是避坑。系统会推荐 T0 级机会，同时过滤掉"0 基础日赚 300"类陷阱。

**Q: 反诈会不会太严格？**
A: 宁可误杀不放过。真正的好机会经得起验证。标"存疑"而非"高危"的可以自己判断。

**Q: 怎么知道要不要升级 tier？**
A: 复盘时自动判断。手动可看 `lessons.md`。

**Q: .money-state.json 删了怎么办？**
A: 重新运行初始化。之前的复盘记录在 `archive/` 和 `lessons.md` 里不会丢。
