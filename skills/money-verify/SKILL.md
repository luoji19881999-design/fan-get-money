# /money-verify — 反诈验证（核心 skill）

这是整套工具最重要的能力：**判断一个机会是不是真能用的。** 宁可错杀，不可放过。

## 跨平台说明（一次性，下文不再重复）

- **资源路径**：下文写的 `../../shared-references/` 是 repo / Claude Code 的路径；**Codex 安装下改读当前 skill 目录的 `references/`**（`install-codex.sh` 已软链进来）。
- **网页检索/打开网页**：Claude Code 用 WebSearch / WebFetch；Codex 用其内置等价能力。`allowed-tools` 是 Claude Code 权限声明，Codex 忽略。

## 唯一标准来源

打分必须严格依据 `../../shared-references/anti-scam-rubric.md`。**不要凭直觉**，逐条对照红线。

## 流程

### Step 1 — 还原机会本质
让用户给出能拿到的信息（名称 / 招募文案 / 链接 / 对方怎么承诺的）。
先回答那句决定性的话：**"谁付钱？为什么付？付多少？"** —— 答不清就已经是 ⚠️ 起步。

### Step 2 — 过硬红线（A1–A6）
逐条对照 rubric A 节。命中任意一条：
- 判定 **高危 ❌**，明确指出命中哪条、为什么。
- 命中 A4（资金/账户经手）或 A6（违法/灰产）→ 额外提醒**可能涉嫌违法犯罪**，劝退。

### Step 3 — 过存疑信号（B1–B6）
对照 rubric B 节，列出命中项。

### Step 4 — 实时查证（rubric C）
**必须实际上网查，不能凭记忆下结论：**
- 网页检索：`<名称> 骗局` / `<名称> 靠谱吗` / `<名称> 被骗 投诉`（检索词加年份）。
- **查实体真伪**：企查查/天眼查搜公司是否真实存在、有无异常；平台是否有备案、是否走担保交易
- **查口碑**：黑猫投诉搜该平台/公司的真实投诉
- 找第三方真实收款/受骗反馈（区分软文与真人）
- **关键数字强制下钻**：任何要写进判定的数字（收入/市场/佣金），必须打开一手原文核对；只来自搜索摘要的，判定里标「未核实」。

### Step 4.5 — 时效核查（rubric C′，强制）
- 查每条关键来源（渠道/价格/玩法）的发布日期。
- 超 24 个月默认失效，不作为"现在怎么做"的依据；12–24 个月标"可能过时"并找近 12 个月来源交叉验证。
- 输出里如实标注信息年份与过时风险。**过时但非骗局的，也要提醒"玩法可能已变，需复核"。**

### Step 5 — 输出（rubric D 格式）
```
机会：<名称>
钱从哪来：<谁付钱/为什么/多少>
硬红线：<命中的 A 编号，或"无">
存疑信号：<命中的 B 编号，或"无">
判定：高危 ❌ / 存疑 ⚠️ / 可行 ✅
验证第一步：<今天就能做、低成本、能确认钱真能到账的动作>
诚实收入预期：<前期多久/多少；说明是时间换小钱还是有快钱>
```
判定规则：命中任一 A → 高危；命中 B 且未查清 → 存疑；无 A、B 查清、钱流向清晰 → 可行。

### Step 6 — 落盘
若已有 `.money-state.json`，把本次结论写/更新到 `opportunities[]`（含 verdict、red_flags、suspect_flags、verify_first_step）。

## 收尾
- 可行 ✅ → 提示用 `money-plan` 做行动方案。
- 存疑 ⚠️ → 给出"验证第一步"，让用户先小成本试，别急着投入。
- 高危 ❌ → 解释清楚坑在哪，附 rubric E 的清醒剂。
- 任何情况都加一句：本结论基于 <今天日期> 的检索，最终请自行复核、风险自担。
